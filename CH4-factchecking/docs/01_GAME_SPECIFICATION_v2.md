## 主要差異（v1 → v2）

| 項目 | v1 | v2 |
|---|---|---|
| 主播角色名 | 咬咬 | **吱吱** |
| 開場流程 | 開始頁 → 前導 → 情境選擇 → 第一關 | **說明頁** → 開始頁 → 前導 → 隨機抽情境 → 第一關 |
| 情境選擇 | 玩家自選 3 個情境 | **隨機抽 1 個**（每次玩一個情境，重玩才換） |
| 第一關按鈕 | 單一「停」按鈕 | **「通過」+「停」雙按鈕** |
| 第一關計時 | 每則停留 3 秒 | **每則 5 秒倒數**，到時視同答錯 |
| 第一關過關條件 | 找出 3 則假訊息 | **2 則假訊息都答對** |
| 證書 | 完成 3 情境 1 張 | **1/2/3 星制**，依完成情境數分級 |

## 角色設定

**主播吱吱**：奶黃色卡通小雞，戴著棕色耳機式麥克風（如 podcaster 造型），脖子有橘色蝴蝶結領結，雙頰有橘色圓形腮紅。風格：手繪繪本風、Q 版圓潤、棕色描邊、可愛親切（類似 LINE 貼圖質感），不寫實。

> 預設立繪：專案根目錄已有 `吱吱260427.png`，是中性站姿，先複製為 5 個檔名給程式預設使用，之後生成新姿勢時保持外型一致即可。

吱吱會在以下場合出現：開始頁、前導動畫、各關開場與通關、答對 / 答錯提示、證書頁。

## 完整遊戲流程

```
[1] 說明頁（3 步驟） → 開始遊戲
        ↓
[2] 開始頁（輸入姓名）
        ↓
[3] 前導動畫（吱吱出場，4 句對白）
        ↓
[4] 隨機抽選一個情境（優先未玩過）
        ↓
[5] 第一關：停（5 秒倒數、雙按鈕）
        ↓
[6] 第二關：查（60 秒、3 個破綻）
        ↓
[7] 第三關：問（3 角色選 1）
        ↓
[8] 情境結算頁
        ↓
   <三個都完成了？>
   ├─ 是 → [9] 三星證書頁
   └─ 否 → 「再來一場」按鈕 → 回到 [4] 隨機抽下一個未玩情境
```

## 各頁面規格

### 頁面 1：遊戲說明頁

說明頁直接放在最前面、第一次進入網站就看到。共 3 步，最後一步按「開始遊戲」進入下一頁。

**畫面元素**：
- 上方：步驟指示器（1 / 3、2 / 3、3 / 3）
- 中央：大圖顯示吱吱的對應立繪（stop / check / ask）
- 立繪旁：步驟標題 + 說明文字
- 下方：「下一步」按鈕（第 3 步換成「開始遊戲」）

**3 步內容**（資料來源：`data/instructions.csv`）：
1. **停** — 看到可疑的訊息，先按下「停」按鈕！
2. **查** — 放大圖片仔細看，找出 3 個破綻！
3. **問** — 選對該問的人，揭穿真相！

**互動**：純線性，沒有跳過選項。每次進入網站都會顯示（不記憶）。

---

### 頁面 2：開始頁

**畫面元素**：
- 遊戲標題：「AI 真相大搜查」
- 副標題：「校園主播危機」
- 玩家姓名輸入框（必填）
- 「開始遊戲」大按鈕
- 背景：操場或校園插圖

**互動**：
- 姓名為空 → 提示「請先輸入你的名字喔！」（從 ui_strings.csv 取）
- 點開始 → 存 `aitruth_player_name` 到 localStorage → 進入前導動畫

---

### 頁面 3：前導動畫（flipbook + 打字機）

**畫面元素**：
- 上方：18 張司令台 / 福利社場景的逐格動畫圖（`assets/images/intro/move1.png` ~ `move18.png`）
- 下方：對話框，文字以打字機效果一字一字浮現

**對話腳本**（資料來源：`data/narrator_lines.csv`，`line_id = intro_full`）：
> 各位同學請注意！我是校園主播吱吱。現在網路上充滿了真假難辨的 AI 資訊，連我們的校園福利社都變到司令台上了！大家快記住『停、查、問』口訣，我們只有 13 分鐘來拯救校園，找出隱藏的 AI 假消息！GO！ GO！

**時序設計**：
- 圖片輪播每 1600ms 換一張，18 張 → 28.8 秒
- 第 7 張時吱吱抵達麥克風前面，**此時打字機才啟動**
- 打字機每字 ~180ms（動態計算，1.5x 原速），從 frame 7 到 slideshow 結束剛好打完
- 兩者**同時收尾**，自然停在 GO! GO! 畫面

**結束後**：
- slideshow 下方出現「🔍 開始調查！」按鈕（不擋對話框）
- 副標：「本次任務：[情境名]」
- 玩家點下按鈕才進入第一關（給足準備時間）

**互動**：動畫期間純播放，無法跳過。結束後等待玩家手動點擊按鈕。「13 分鐘來拯救校園」只是台詞氣氛，並非真實全域計時。

---

### 頁面 4：隨機抽選情境（背景邏輯，無 UI）

**邏輯**：
```javascript
const completed = JSON.parse(localStorage.getItem('aitruth_completed_scenarios') || '[]');
const allScenarios = ['s1', 's2', 's3'];
const remaining = allScenarios.filter(id => !completed.includes(id));
const pool = remaining.length > 0 ? remaining : allScenarios;
const chosen = pool[Math.floor(Math.random() * pool.length)];
```

優先抽未玩過的，三個都玩過則純隨機。

抽完後 fade-in 顯示「事件：[情境名稱]」3 秒，然後進入第一關。

---

### 頁面 5：第一關「停」（約 1 分鐘）

**畫面元素**：
- 上方：本情境的標題 + 進度條（已答對的假訊息數）
- 中央：訊息顯示區（圖片 + 文字）
- 下方左：「✅ 通過」按鈕（綠色）
- 下方右：「⛔ 停！」按鈕（紅色）
- 右上角：5 秒倒數計時器（圓形進度條）

**遊戲邏輯**：
- 依序播放 5 則訊息
- 每則訊息顯示後，玩家**最多 5 秒**內必須點擊「通過」或「停」
  - 5 秒內點對 → 慶祝動畫 → 下一則
  - 5 秒內點錯 → 鼓勵性提示「別緊張…」→ 下一則（**不重複**）
  - 5 秒到還沒按 → 視同答錯 → 提示「時間到！下次再快一點～」→ 下一則
- 最後 2 秒倒數音效會加速，營造緊張感

**慶祝動畫分級**：
| 情況 | 視覺 | 音效 |
|---|---|---|
| 點對真訊息（按通過） | 綠色光圈 1 次 + 「✓」icon 彈跳 | 輕柔「叮」 |
| 點對假訊息（按停） | **紅黃彩帶噴發 + 獎章彈跳放大 + 螢幕邊框閃光** | **較強的慶祝音** |

**過關條件**：5 則訊息結束後，**2 則假訊息（is_fake=true）都被點對「停」** 即過關。

**未過關**：第一關結束後若兩則假訊息沒全部答對，回到開始頁可重玩（不算完成此情境）。

**獎勵**：
- 🏆 第 1 個獎盃
- 該情境的主圖解鎖（給第二關用）

---

### 頁面 6：第二關「查」（約 1 分鐘）

**畫面元素**：
- 上方：標題「找出 3 個破綻」+ 60 秒倒數
- 中央：主圖（1600×900，響應式縮放）
- 圖上點擊後出現紅色圓圈標記 + 動畫
- 右上角：破綻計數（已找到 0/3）
- 右下角：「需要提示」按鈕（限用 1 次）

**遊戲邏輯**：
- 玩家在主圖上點擊
- 點擊位置與 hotspot 中心點距離 ≤ `tolerance_pct%`（依百分比計算）→ 命中
- 命中：紅圈標記 + 彈出說明卡（破綻名稱 + 為什麼是破綻）+「叮」音效
- 沒命中：輕微震動，**無懲罰**
- 點「需要提示」：尚未找到的某個破綻位置短暫脈衝 2 次
- 60 秒到 → 進入下一關（已找到的破綻保留，沒找到的會在第三關提示）

**過關條件**：60 秒內找齊 3 個破綻（找不齊也會強制進入第三關）

**獎勵**：
- 🏆 第 2 個獎盃
- 3 個破綻存入「證據庫」，第三關會顯示

---

### 頁面 7：第三關「問」（約 0.5 分鐘）

**畫面元素**：
- 上方：第二關找到的破綻列表（小卡片橫排）
- 中段：問題文字「根據以下破綻，應該問誰最能確認真假？」
- 下方：3 個角色卡（頭像 + 顯示名稱），平均分布

**遊戲邏輯**：
- 玩家點擊任一角色 → 該角色出現大頭像 + 對話框，播放對白
- 正確角色（`is_correct=true`）：完整對白 + 通關慶祝
- 錯誤角色：對白 + 提示「再想想吧」+ 角色卡淡化 + 可重選
- 答錯不扣分、不扣時間

**過關條件**：選對正確角色

**獎勵**：
- 🏆 第 3 個獎盃
- 該情境完成 → 加入 `aitruth_completed_scenarios`

---

### 頁面 8：情境結算頁

**畫面元素**：
- 標題：「[情境名稱] 完成！」
- 「你的調查紀錄」故事樹：
  - 第一關：你找到的假訊息（縮圖）
  - 第二關：你發現的 3 個破綻（縮圖橫排）
  - 第三關：你選的角色 + 真相結論
- 右下：「前往證書」按鈕

**互動**：點「前往證書」進入頁面 9。

---

### 頁面 9：證書頁

**畫面元素**：
- 證書設計（含玩家姓名、頒發日期）
- 星數顯示（⭐ / ⭐⭐ / ⭐⭐⭐ 依完成情境數）
- 稱號：
  - 1 個情境完成 → 「⭐ 一星探員」
  - 2 個情境完成 → 「⭐⭐ 二星偵探」
  - 3 個情境完成 → 「⭐⭐⭐ 三星偵探（最高榮譽）」
- 吱吱-加油 立繪 + 對應旁白（`cert_1star` / `cert_2star` / `cert_3star`）
- 按鈕區：
  - 「下載證書」（PNG，html2canvas）
  - 若未滿三星 → 「再挑戰一場」（隨機抽未玩情境）
  - 若已三星 → 「重新挑戰」（清空 localStorage，重新開始）

## 技術需求

### 必要功能
- 響應式設計：**平板優先（iPad 橫向 4:3）**，PC 為輔
- 觸控優化：按鈕最小 48×48 px
- 進度儲存：localStorage（玩家姓名、已完成情境、星數）
- 證書下載：html2canvas → PNG
- 音效：背景音樂（可關閉）、按鈕音、慶祝音、倒數音
- 配音：吱吱旁白與角色對白播放

### 圖片規格
- 主圖：1600×900 PNG / JPG（16:9）
- 訊息配圖：800×600 PNG / JPG（4:3，內含模擬手機 / 新聞畫面）
- 吱吱立繪：800×1000 PNG 透明背景（3:4）
- 角色立繪：800×1000 PNG 透明背景（3:4）
- 所有圖片壓縮到 500KB 以下

### 座標系統
- 所有 hotspot 座標一律使用**百分比（0-100）**
- 點擊位置先換算成百分比再判定，支援任何螢幕尺寸
- 點擊容錯：`distance² ≤ tolerance_pct²`

### 瀏覽器支援
- Chrome / Safari / Edge 最新版（桌機 + 平板）
- 不支援 IE

### 部署
- 開發：VS Code Live Server
- 發布：GitHub Pages（HTTPS、fetch CSV 沒問題）

### 不需要的功能
- ❌ 後端伺服器
- ❌ 用戶帳號系統
- ❌ 線上排行榜
- ❌ 多人對戰

## 視覺風格

詳見 `style_demo.html`。簡述：

- **色彩**：明亮、克制（深藍 #2E5C8A + 橙 #FF8C42 為主，避免大量漸層）
- **字體**：Noto Sans TC / 微軟正黑體
- **角色設計**：兒童繪本卡通風
- **整體氛圍**：偵探故事 + 校園溫馨，但不是 dark mode、不是科幻發光感

### 禁止的視覺元素
- 大範圍 blur 毛玻璃
- 紫粉藍漸層
- 霓虹發光
- 半透明卡片堆疊

### 鼓勵的視覺元素
- 實心色塊 + 細邊框
- 按鈕按下有實體位移感
- 慶祝動畫用彩帶 / star burst（SVG），不用光暈

---

## 不能省略的細節

1. 每個按鈕都有點擊反饋（變色、位移、音效）
2. 過關都有慶祝動畫
3. 答錯不打擊（鼓勵性語句）
4. 載入畫面要有趣（吱吱動畫，不要白屏）
5. 文字大小可調、按鈕對比清楚（無障礙）

## 開發步驟（給 Cowork 實作）

1. 確認 `data/` 8 份 CSV 與 `docs/IMAGE_REQUIREMENTS.md`
2. 完成 `style_demo.html`（樣式範例頁，視覺方向定調）
3. 建立 `index.html` 主架構（所有 screen 的空 section）
4. 建立 `styles.css` 引入 CSS 變數
5. 建立 `script.js` 基本 CSV 載入 + screen 切換
6. 依序實作：說明頁 → 開始頁 → 前導 → 第一關 → 第二關 → 第三關 → 結算 → 證書
7. 整合圖片 / 配音 / 音效
8. 平板 + PC 測試
9. 部署到 GitHub Pages

每完成一步請開發者確認後再走下一步。

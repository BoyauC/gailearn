## 1. 專案一句話描述

15 分鐘 web 互動遊戲，國小三~六年級學生用「停、查、問」三步驟訓練辨識 AI 假消息。主角是校園小主播「吱吱」（奶黃色擬人化小雞）。

---

## 2. 技術架構

### 2.1 技術棧

| 項目 | 選擇 | 理由 |
|---|---|---|
| 前端 | 原生 HTML + CSS + JavaScript（無框架） | 一個人維護、最終可雙擊 index.html 即玩、不用 build 工具 |
| 資料 | CSV（用 fetch 載入） | 非工程師也能用 Excel/Google Sheets 編輯題庫 |
| 圖片 | WebP（quality 90） | 比 PNG 省 87% 流量，現代瀏覽器全支援 |
| 進度儲存 | localStorage | 無後端、無帳號 |
| 證書下載 | html2canvas（CDN） | DOM 轉 PNG |
| 部署 | GitHub Pages | HTTPS、免費、`fetch` CSV 沒問題 |

## 3. 完整遊戲流程（9 階段）

```
[1] 單頁式開始頁
    └─ 標題 + 玩法說明（停/查/問 3 步驟）+ 小提醒 + 開始按鈕
        ↓ 玩家按「開始」
[2] 前導動畫
    └─ 18 張司令台/福利社場景 flipbook（每張 1600ms，共 28.8 秒）
    └─ 第 7 張吱吱抵達麥克風時，打字機啟動
    └─ 對白：「注意！我是校園主播吱吱…GO！ GO！」
        ↓ 玩家按「🔍 開始調查！」按鈕
[3] 隨機抽情境（背景邏輯，無 UI）
    └─ 從未玩過的情境中隨機 1 個（全玩過則純隨機）
        ↓
[4] 情境揭幕
    └─ 全螢幕顯示「事件來了！[情境名稱]」2.5 秒
        ↓ 自動進入
[5] 第一關「停」（約 1 分鐘）
    └─ 5 則訊息逐一輪播，每則 5 秒倒數
    └─ 玩家選「✅ 通過」或「⛔ 停！」
    └─ 過關條件：2 則假訊息都答對「停」
    └─ 視覺反饋：
        ├─ 真訊息答對 → 綠光圈 + 勾勾（輕量慶祝）
        └─ 假訊息答對 → 紅黃彩帶 + 獎章 + 邊框閃光（強烈慶祝）
    └─ 失敗 → 跳過前導，重抽情境再試（不算完成）
        ↓ 成功
[6] 第二關「查」（60 秒）
    └─ 從本情境的 2 張假訊息中【隨機抽 1 張】當主圖
    └─ 對應該假圖的 3 個破綻 hotspot 出現可點擊
    └─ 玩家在主圖點擊找出 3 個破綻
    └─ 命中：紅圈標記 + 破綻說明卡 + 輕量慶祝
    └─ 提示功能限用 1 次
    └─ 60 秒到 → 強制進下一關（找到的破綻保留）
        ↓
[7] 第三關「問」（無計時）
    └─ 顯示第二關找到的破綻列表（chip）
    └─ 3 個角色卡橫排（順序每次隨機）
    └─ 玩家選對應的「該問的人」
    └─ 答對：完整對白 + 強烈慶祝 → 進結算
    └─ 答錯：該卡淡化、提示再想想，可重選
        ↓
[8] 結算頁
    └─ 故事樹 3 段回顧：
        ├─ 第一關 你抓到的假訊息（列出 text）
        ├─ 第二關 你找到的破綻（含標題）
        └─ 第三關 你問對了人（角色名）
    └─ 按「前往證書」
        ↓
[9] 證書頁
    └─ ⭐⭐⭐ 三星偵探（最高榮譽）— 每完成 1 情境即得
    └─ 證書內容：調查任務（情境名）+ 持有人 + 日期
    └─ 姓名輸入欄（選填，即時更新）
    └─ 訊息：
        ├─ 完成 < 3 情境 → 「總共有 3 個情境可以練習喔！」
        └─ 完成 3 情境 → 「🎉 太棒了！你已挑戰完三個情境！」
    └─ 按鈕：
        ├─ 下載證書（PNG，html2canvas）
        ├─ 再挑戰一場（< 3 時顯示，跳過前導直接抽下一個）
        └─ 重新開始（= 3 時顯示，清空進度回開始頁）
```

## 7. 已知議題與設計決策

### 7.1 第二關隨機假圖機制（v3 新加）

每情境有 **2 張假訊息圖**（各 3 個破綻 = 6 破綻 / 情境），第二關開始時：

```javascript
// script.js startLevel2() 邏輯
const fakeMessages = level1.messages.filter(m => m.is_fake);
const pickedFake = fakeMessages[Math.floor(Math.random() * fakeMessages.length)];
const breakpoints = data.breakpoints.filter(b =>
  b.scenario_id === sc.id && b.source_message_order === pickedFake.message_order
);
```

**效果**：玩家重玩同一情境時，第二關 50% 機率看到不同的假圖 + 不同的破綻組合，提升再玩性。

### 7.2 證書系統

每完成 1 個情境就拿一張 **三星證書**（最高榮譽）。證書上顯示**該次完成的情境名稱**。完成所有 3 個情境後訊息會變成「🎉 你已挑戰完三個情境！」並顯示「重新開始」按鈕。

### 7.3 重玩流程

| 觸發 | 行為 |
|---|---|
| 第一關失敗 | 自動跳過前導，重抽情境（優先未玩過） |
| 證書頁「再挑戰一場」 | 同上，跳過前導 |
| 證書頁「重新開始」 | 清空 localStorage、回開始頁、再走完整 intro |
| 玩家直接重整網頁 | 載入 localStorage 進度，下一個情境照常隨機 |

### 7.4 圖片格式

全專案統一 **WebP**（quality 90）。從 PNG 換 WebP 後 intro 動畫從 40MB 縮到 5MB（省 87%），上 GitHub Pages 也不會超出流量配額。

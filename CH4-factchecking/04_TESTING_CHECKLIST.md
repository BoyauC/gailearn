# 04 — 測試清單

## 測試流程

完成程式開發後，請依照以下檢查表逐項測試。

---

## A. 功能測試（必須全部通過）

### A1. 開始頁
- [ ] 玩家可輸入姓名
- [ ] 姓名空白時無法開始（顯示提示）
- [ ] 點擊「開始」進入前導動畫
- [ ] 姓名正確儲存到 localStorage

### A2. 前導動畫
- [ ] 對話逐句顯示
- [ ] 「下一步」按鈕可推進
- [ ] 動畫流暢，無卡頓
- [ ] 完成後進入情境選擇

### A3. 情境選擇
- [ ] 3 個情境卡片正確顯示
- [ ] 點擊任一情境進入第一關
- [ ] 已完成的情境顯示「✅」標記

### A4. 第一關：停
- [ ] 5 則訊息依序播放
- [ ] 每則停留 3 秒
- [ ] 倒數計時器正確
- [ ] 點擊「停」有反應
- [ ] 點擊正確的假訊息：計數 +1
- [ ] 點擊錯誤的真訊息：有提示
- [ ] 找到 3 則假訊息後過關
- [ ] 過關有慶祝動畫
- [ ] 獎盃計數正確

### A5. 第二關：查
- [ ] 圖片正確顯示
- [ ] 點擊 hotspot 位置正確（容錯 ±20px）
- [ ] 點擊正確破綻：顯示說明卡
- [ ] 點擊錯誤位置：無懲罰
- [ ] 「提示」按鈕可使用 1 次
- [ ] 找齊 3 個破綻後過關
- [ ] 90 秒倒數計時正確

### A6. 第三關：問
- [ ] 上方顯示第二關找到的 3 個破綻
- [ ] 3 個角色選項可點擊
- [ ] 點擊角色：播放對應回應（文字+語音）
- [ ] 選對：過關
- [ ] 選錯：可重新選擇
- [ ] 過關後獲得第 3 個獎盃

### A7. 結算與證書
- [ ] 結算頁顯示故事樹
- [ ] 「下一個情境」按鈕可用
- [ ] 3 情境全完成後顯示證書頁
- [ ] 證書包含玩家姓名與日期
- [ ] 「下載證書」功能正常（PNG）

---

## B. 視覺測試

### B1. 桌機（1920x1080）
- [ ] 所有元素正確排版
- [ ] 文字清楚可讀
- [ ] 按鈕大小適中
- [ ] 圖片不變形

### B2. 平板（1024x768，iPad）
- [ ] 響應式佈局正確
- [ ] 觸控按鈕足夠大
- [ ] 沒有橫向滾動條
- [ ] 圖片載入流暢

### B3. 視覺效果
- [ ] 場景切換有淡入淡出動畫
- [ ] 按鈕點擊有視覺反饋
- [ ] 過關有慶祝特效
- [ ] 沒有閃爍或抖動

---

## C. 音效測試

- [ ] 開始遊戲後背景音樂播放
- [ ] 「靜音」按鈕功能正常
- [ ] 按鈕點擊有音效
- [ ] 過關有慶祝音效
- [ ] 答對/答錯有對應音效
- [ ] 角色配音清晰，音量適中
- [ ] 倒數最後 3 秒有提示音

---

## D. 瀏覽器相容性

請在以下瀏覽器測試：

- [ ] Chrome（桌機）
- [ ] Safari（桌機 Mac）
- [ ] Edge（桌機 Windows）
- [ ] Safari（iPad）
- [ ] Chrome（Android 平板）

每個瀏覽器都需測試：
- 完整玩通 1 個情境
- 證書下載功能
- 音效播放

---

## E. 邊界情況測試

### E1. 不尋常的操作
- [ ] 玩家在訊息播放中快速連點：不會出錯
- [ ] 玩家在第二關不點任何位置直到時間到：給予適當提示
- [ ] 玩家在第三關不選擇就重新整理頁面：進度正確恢復
- [ ] 玩家輸入超長姓名（30 字以上）：自動截斷或限制

### E2. 網路與效能
- [ ] 慢速網路下圖片載入：有載入動畫
- [ ] 大圖片是否壓縮過：每張不超過 500KB
- [ ] 總資源大小：不超過 50MB

### E3. 容錯
- [ ] 圖片載入失敗：顯示替代文字
- [ ] 音效檔案缺失：不會卡住遊戲
- [ ] localStorage 被禁用：仍可玩，但進度不儲存

---

## F. 教學效果驗收（如有條件）

### F1. 找 3-5 位實際小學生試玩
- [ ] 能否在無協助下完成 1 個情境
- [ ] 完成後能否說出「停、查、問」的意思
- [ ] 能否舉例「什麼樣的訊息要小心」
- [ ] 對遊戲的喜愛程度（1-5 分）

### F2. 家長/老師反饋
- [ ] 教學內容是否清楚
- [ ] 難度是否適合三至六年級
- [ ] 是否有不當內容
- [ ] 是否願意推薦給其他學生

---

## 記錄問題的格式

當發現問題時，請開發者用此格式記錄：

```
## 問題編號：BUG-001
### 發生位置
- 情境 2、第一關
### 問題描述
- 點擊「停」按鈕後，計數沒有增加
### 重現步驟
1. 進入情境 2
2. 等待第 3 則訊息出現
3. 點擊「停」按鈕
### 預期結果
- 計數從 0 變為 1
### 實際結果
- 計數仍為 0
### 優先級
- 高（影響核心玩法）
```

---

## 測試完成標準

| 等級 | 條件 |
|------|------|
| ✅ 可發布 | A 全部通過 + B/C 90%+ 通過 + D 至少 3 個瀏覽器通過 |
| ⚠️ 需修改 | A 有 1-2 項未通過 |
| ❌ 不可發布 | A 有 3+ 項未通過 或 D 主要瀏覽器無法執行 |

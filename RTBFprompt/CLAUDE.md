# RTBF 提示詞拼圖遊戲 — 開發規格文件

## 專案概述

**名稱**：RTBF 提示詞拼圖挑戰  
**用途**：國中 AI 素養課程教學遊戲  
**裝置**：平板觸控操作為主  
**目標**：讓學生透過拼圖遊戲，理解 RTBF 四個元素組成有效 AI 提示詞的邏輯  
**時長**：約 15–20 分鐘（四關）  
**檔案格式**：單一 HTML 檔案，不依賴外部框架或資料庫

---

## 教學核心概念

### RTBF 四元素定義

| 元素 | 英文 | 中文 | AI 的工作 | 學生問自己 |
|------|------|------|-----------|-----------|
| R | Role | 角色 | AI 扮演誰 | 我希望 AI 像誰一樣回答我？ |
| T | Task | 任務 | AI 要做什麼 | 我要 AI 幫我「做什麼」？ |
| B | Background | 背景 | 你的情況 | AI 需要知道我哪些「狀況與需求」？ |
| F | Format | 格式 | 怎麼呈現 | 我希望答案「長什麼樣子」？ |

### 教學設計邏輯

- 每關對比「❌ 模糊問法」與「✅ RTBF 精準問法」，讓學生感受差異
- 遊戲機制強迫學生主動判斷每個元素的最佳選項
- 解鎖後動畫逐句呈現完整精準提示詞，加深印象

## 遊戲機制規格

### 核心玩法

1. 四個方塊（R、T、B、F）散落在畫面各處
2. 玩家**旋轉90度**，方塊上方文字隨之切換
3. 玩家**拖移方塊**至底部箱子區域，網格吸附定位
4. 四個方塊填滿 5×3 格子且內容符合標準答案 → 過關

### 過關條件（需同時滿足）

- **條件一｜形狀正確**：四個方塊恰好填滿 5×3 = 15 格，不留空格
- **條件二｜內容正確**：每個方塊旋轉到的那一面文字，必須是該關卡的標準答案（嚴格版，唯一正解）

### 錯誤處理

- 形狀填滿但內容有誤 → 錯誤方塊變紅色閃爍 + 提示文字說明哪個方塊需調整
- 玩家不需重新拖移，原地點擊旋轉到正確方向即可
- 形狀放不進去（超出邊界或重疊）→ 方塊彈回原位

### 操作方式

| 操作 | 效果 |
|------|------|
| 點擊方塊 | 旋轉90度，切換顯示文字 |
| 長按拖移 | 移動方塊位置 |
| 拖入箱子範圍 | 自動吸附到最近網格 |
| 點擊「💡 提示」按鈕 | 顯示本關提示浮層 |

### 各方塊格數與形狀

## 技術實作規格

### 檔案結構
- **單一 HTML 檔案**（包含 CSS + JavaScript）
- 不依賴外部 JavaScript 框架（React、Vue 等）
- 不需要後端或資料庫
- 可直接在瀏覽器開啟或部署至 GitHub Pages

### 關鍵技術要點

**拖移實作**
- 使用 Pointer Events API（`pointerdown`、`pointermove`、`pointerup`）
- 同時支援觸控與滑鼠
- 設定 `touch-action: none` 避免瀏覽器預設手勢衝突

**網格吸附邏輯**
```
1. 取得方塊中心點座標
2. 計算與箱子網格各格的距離
3. 找出距離最近且未被佔用的合法位置
4. 判斷整個方塊形狀是否在5×3範圍內
5. 若合法 → 吸附；若超出或重疊 → 返回原位
```

**過關判定流程**
```
1. 檢查四個方塊是否全部已放入箱子（placed = true）
2. 計算箱子內佔用格數是否等於15
3. 檢查每個方塊當前旋轉方向 = 標準答案方向（rotation % options.length === correctIndex）
4. 全部通過 → 觸發成功動畫
5. 有誤 → 標記錯誤方塊，顯示提示
```

**旋轉文字對應**
- B 方塊有2種旋轉 → 對應2個選項
- R、T、F 方塊有4種旋轉 → 對應4個選項
